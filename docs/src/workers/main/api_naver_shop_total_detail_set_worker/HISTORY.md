# 변경 이력

## 2026-07-14

### V_105

#### 수정

* DB 직접 처리 코드를 `WorkerDbRepository` 방식으로 변경
* 작업 이력, 상세 데이터 저장, 성공·실패 건수 집계를 Repository에서 처리하도록 수정
* 프로그램 종료 시 DB 작업 이력 마감 및 엑셀 자동 저장 처리 추가
* 네이버 쇼핑 접속 제한 페이지 감지 기능 추가

    * `쇼핑 서비스 접속이 일시적으로 제한되었습니다`
    * `해당 네트워크의 접속을 일시적으로 제한`
    * `비정상적인 접근이 감지`

* 접속 제한 감지 시 브라우저를 종료하고 12시간 대기하도록 수정
* 12시간 후 동일 키워드와 동일 페이지를 다시 확인하도록 수정
* 접속 제한이 유지되면 추가로 12시간 대기하도록 수정
* 총 24시간 대기 후에도 접속 제한이 유지되면 작업을 실패 처리하고 종료하도록 수정
* 접속 제한 상태에서는 다음 페이지로 이동하거나 진행률을 증가시키지 않도록 수정
* 장시간 대기 중에도 사용자가 정지할 수 있도록 중단 감지 방식으로 수정
* 접속 제한 및 캡차 최종 실패 시 작업 결과가 정상 성공으로 표시되지 않도록 반환값 수정

#### 결정사항

* 고객이 동일 키워드를 반복 입력해 새로 노출되는 업체를 수집할 수 있으므로 키워드 중복은 제거하지 않음
* 기존 DB와 현재 실행에서 수집한 스토어명을 기준으로 업체 중복 제거 유지
* 접속 제한 발생 시 짧은 간격으로 반복 요청하지 않고 최대 24시간 동안 두 차례만 재확인
* 접속 제한 해제 후 실패했던 키워드와 페이지부터 작업 재개
* 24시간 이후에도 제한이 해제되지 않으면 자동 반복하지 않고 작업 종료

#### 테스트

* Repository 초기화 및 작업 이력 생성 확인
* 상세 데이터 저장 확인
* DB 데이터 기준 엑셀 저장 확인
* 접속 제한 문구 감지 확인
* 접속 제한 시 현재 페이지 유지 확인
* 접속 제한 최종 실패 시 `FAIL` 처리 확인
* 사용자 정지 시 `STOP` 처리 확인
