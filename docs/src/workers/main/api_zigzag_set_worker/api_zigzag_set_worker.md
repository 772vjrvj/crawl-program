## 2. 작업 목적

지그재그 의류 카테고리의 상품을 조회하고, 각 상품 상세정보에서 판매자 정보를 수집한다.

상품을 기준으로 상세정보를 조회하지만, 최종 결과는 **판매자 기준으로 한 번만 저장**한다.

기존 구글 스프레드시트에 저장된 판매자와도 비교하여 신규 판매자만 추가할 수 있도록 구성한다.

다음 항목은 수집 결과 확인과 오류 추적을 위해 내부적으로 함께 저장한다.

따라서 URL과 상품ID는 내부 DB에는 저장할 수 있지만, 고객에게 전달하는 엑셀이나 결과에서는 제외할 수 있다.

## 5. 판매자 중복 기준

판매자 중복키는 다음 세 항목을 조합하여 생성한다.

```text
업체명 + 대표자 + 전화번호
```

비교 전에 다음과 같이 데이터를 정리한다.

* 업체명: 앞뒤 공백 제거
* 업체명: 연속된 공백 정리
* 대표자: 앞뒤 공백 제거
* 전화번호: 하이픈과 공백 제거
* 전화번호: 숫자만 남겨 비교

예시:

```text
(주)테스트 | 홍길동 | 010-1234-5678
(주)테스트 | 홍길동 | 01012345678
```

위 두 값은 같은 판매자로 처리한다.

### 카테고리 중복 처리

카테고리와 관계없이 판매자 중복키가 같으면 최초로 발견된 한 건만 저장한다.

```text
상의에서 판매자 A 발견
→ 저장

팬츠에서 판매자 A 재발견
→ 중복 제외
```

따라서 최초로 수집된 카테고리와 대표 상품 정보가 최종 결과에 남는다.

## 6. 상품 중복과 판매자 중복

중복 제거는 다음 순서로 처리한다.

### 상품 중복 제거

상품ID 또는 상품 URL을 기준으로 같은 상품을 다시 상세조회하지 않는다.

### 현재 실행 중 판매자 중복 제거

이번 실행에서 이미 확인한 판매자 중복키가 있으면 저장하지 않는다.

### 기존 구글 시트 판매자 중복 제거

프로그램 시작 시 구글 시트 데이터를 조회한다.

기존 시트에 존재하는 판매자 중복키와 같으면 신규 결과에서 제외한다.

전체 흐름은 다음과 같다.

```text
상품 목록 조회
→ 상품ID 또는 URL 중복 확인
→ 상품 상세정보 조회
→ 판매자 정보 추출
→ 판매자 중복키 생성
→ 이번 실행 중복 확인
→ 기존 구글 시트 중복 확인
→ 신규 판매자만 저장
```

## 7. 페이지 수집 기준

지그재그 목록은 화면에서 스크롤할 때 상품이 계속 추가되는 구조다.

프로그램에서는 스크롤 한 번에 추가되는 상품 묶음을 한 페이지로 계산한다.

* 스크롤 1회 분량: 1페이지
* 페이지당 상품 수: 약 50개
* 설정된 시작 페이지부터 종료 페이지까지 수집
* 선택한 카테고리마다 동일한 페이지 범위 적용

예시:

```text
시작 페이지: 1
종료 페이지: 10
페이지당 약 50개
카테고리별 최대 약 500개 상품 조회
```

이 페이지 처리 기준은 고객에게 별도로 설명한다.

실제 상품 수는 지그재그 응답 상태와 마지막 페이지 데이터 수에 따라 달라질 수 있다.

## 8. 프로그램 처리 순서

```text
1. 프로그램 설정값 조회
2. 구글 스프레드시트 기존 데이터 조회
3. 기존 판매자 중복키 목록 생성
4. 선택한 카테고리 목록 순회
5. 시작 페이지부터 종료 페이지까지 상품 목록 조회
6. 상품ID 또는 URL 기준 상품 중복 제거
7. 상품 상세정보 조회
8. 판매자 정보 추출
9. 업체명 + 대표자 + 전화번호로 판매자 중복키 생성
10. 현재 실행 중 판매자 중복 제거
11. 기존 구글 시트 판매자 중복 제거
12. 신규 판매자만 로컬 DB에 저장
13. 설정에 따라 엑셀 파일 저장
14. 신규 판매자를 구글 스프레드시트 마지막 행에 추가
```

## 9. 구글 스프레드시트 연동

프로그램 설정에서 다음 정보를 사용한다.

* 스프레드시트 ID
* 워크시트명
* 서비스 계정 인증 JSON 경로

### 초기 데이터 조회

프로그램 시작 시 구글 시트 데이터를 가져온다.

가져온 기존 데이터에서 판매자 중복키를 생성하여 메모리에 보관한다.

### 신규 데이터 추가

크롤링 완료 후 기존 시트에 없는 신규 판매자만 구글 시트의 마지막 데이터 아래에 추가한다.

### 초기 개발 순서

초기 크롤링 테스트 단계에서는 다음 코드를 작성한 뒤 주석 처리한다.

* 구글 시트 기존 데이터 조회
* 구글 시트 신규 데이터 추가

구글 시트 조회가 비활성화된 동안에는 기존 데이터가 없는 것으로 처리한다.

```python
existing_seller_keys = set()
```

따라서 이 상태에서는 이번 실행 중 발생한 판매자 중복만 제거하고, 나머지는 모두 신규 데이터로 처리한다.
