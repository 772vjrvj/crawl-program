# launcher_window.py

## 역할

GB7 Launcher의 메인 화면을 구성하고 다음 기능의 전체 흐름을 제어한다.

- 긴급 공지 확인
- 프로그램 업데이트 확인
- 새 버전 다운로드 및 실행
- 현재 버전 실행
- 상태·진행률·로그 표시
- 공식 사이트 및 문의 페이지 연결

---

## 실행 흐름

```text
런처 화면 생성
    ↓
현재 프로그램 정보 확인
    ↓
지원 링크 설정
    ↓
긴급 공지 확인
    ↓
업데이트 확인
    ↓
업데이트 선택 또는 현재 버전 실행
```

---

## 주요 화면 구성

### 상태 영역

현재 처리 상태를 사용자에게 표시한다.

- 초기화
- 긴급 공지 확인
- 업데이트 확인
- 업데이트 진행
- 준비 완료
- 실패

진행률은 `QProgressBar`로 표시하고 상세 내용은 상태 문구와 로그에서 확인한다.

### 긴급 공지

서버에서 긴급 공지를 조회한다.

- 긴급 공지가 있을 때만 다시 보기 버튼 표시
- 오늘 하루 안보기 상태 관리
- 공지 처리가 끝나면 업데이트 확인 진행

### 지원 센터

지원 영역은 클릭시 해당 사이트로 이동한다.

### 하단 버튼

- 로그 보기 / 숨기기
- 재시도
- 실행
- 닫기

업데이트 작업 중에는 실행·재시도·닫기 상태를 제한한다.

---

## 업데이트 처리

업데이트 조회 결과에 따라 다음과 같이 처리한다.

### 최신 버전인 경우

현재 실행 파일을 실행할 수 있는 준비 상태로 전환한다.

### 새 버전이 있는 경우

사용자에게 다음 선택을 제공한다.

- 지금 업데이트
- 현재 버전 실행
- 선택창 닫기

### 업데이트 완료

새 버전 설치가 끝나면 해당 실행 파일을 자동 실행하고 런처를 종료한다.

### 업데이트 실패

기존 실행 파일이 있으면 현재 버전을 실행할 수 있도록 유지한다.

- 실행 버튼 활성화
- 재시도 버튼 활성화
- 오류 메시지와 로그 제공

---

## Worker 연결

화면이 멈추지 않도록 서버 통신과 업데이트 작업은 별도 스레드에서 처리한다.

```text
NoticeWorker
    긴급 공지 조회

UpdateWorker
    버전 확인
    다운로드
    설치
    실행 파일 준비
```

Worker의 상태, 로그, 진행률, 완료 결과는 Qt Signal로 화면에 전달한다.

## 핵심 원칙

- 메인 화면은 UI 표시와 전체 작업 흐름을 담당한다.
- 서버 통신과 파일 작업은 Worker에서 처리한다.
- 업데이트 실패 시에도 가능한 경우 기존 버전을 실행할 수 있게 한다.
- 긴급 공지와 업데이트 확인은 순서대로 처리한다.
- 지원 링크는 설정 파일을 통해 관리한다.
