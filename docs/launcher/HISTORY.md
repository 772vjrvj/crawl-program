* `app.json`

  * 문의 및 Q&A 주소

### 결정사항

* 런처 관련 명령은 프로젝트 최상위 경로에서 실행

* 로컬 실행은 모듈 방식으로 처리

  ```powershell
  python -m launcher.launcher_main
  ```

* 런처 빌드는 프로젝트 최상위의 PowerShell 스크립트로 처리

  ```powershell
  .\build_launcher.ps1
  ```

* 런처는 ZIP 다운로드 URL을 직접 생성하지 않음

* 실제 ZIP 주소는 최신 버전 API 응답의 `asset.url`을 사용

* Cloudflare R2 적용 시 서버가 반환하는 `asset.url`만 변경

* 다운로드 파일은 `.part` 임시 파일로 저장한 뒤 완료 시 ZIP 파일로 변경

* 압축 해제는 `_staging` 폴더에서 먼저 진행

* ZIP 내부의 `CrawlProgram.exe` 존재 여부를 확인한 뒤 정식 버전 폴더로 반영

* 업데이트 설치와 `current.json` 갱신이 완료된 후 이전 버전을 정리

* 고객 PC에는 최신 버전과 직전 버전 총 2개만 유지

* 업데이트 실패 시 현재 사용 중인 버전은 삭제하거나 변경하지 않음

* 사용자가 `지금 업데이트`를 선택하면 설치 완료 후 새 버전을 자동 실행

* 사용자가 `현재 버전 실행`을 선택하면 기존 버전을 즉시 실행

* 업데이트 팝업을 닫거나 `Esc`를 누르면 자동 실행하지 않고 런처 화면 유지

* 오류가 발생하면 자동 실행하지 않고 사용자가 재시도 또는 기존 버전 실행을 선택

* 프로그램 실행 후 런처는 종료하고 실제 프로그램은 독립 프로세스로 유지

* 고객 설정, DB, 로그 등의 데이터는 버전 폴더와 분리하여 관리

### 실행 흐름

```text
런처 실행
    ↓
스플래시 화면 표시
    ↓
긴급 공지 확인
    ↓
최신 버전 확인
    ↓
현재 버전과 최신 버전 비교
    ↓
현재 버전 실행 또는 업데이트 진행
    ↓
프로그램 실행
    ↓
런처 종료
```
