ERP 서비스 구조도

구축형/구독형/커스텀/요금제/중앙관리 정책을 한눈에 볼 수 있도록 정리한 구조도입니다.

1. 요구사항 목록

코드 분류 항목 내용
SYS-001 서비스구조 구축형 지원 [구독형] + [커스텀] 조합으로 지원
SYS-002 서비스구조 구독형 지원 업체 공통 기능을 기반으로 하고, 요금제에 따라 기능을 제어
SYS-004 서비스구조 요금제 Standard / Premium / Premium+ 에 따라 기능, 할당량, 메뉴 접근 범위를 다르게 제어
SYS-005 서비스구조 커스텀 정책 업체별 커스텀 기능 추가 지원
SYS-006 서비스구조 중앙관리 여러 업체 ERP의 주요 변경 데이터를 중앙 서버에서 통합 관리. 업체 ERP에서 CUD 발생 시 중앙 서버 API로 변경 이력을 전송
SYS-007 서비스구조 업체별 관리 업체 구분용 식별자(company_id / server_id) 기반 운영

중앙 관리 / 업체별 관리 구조
여러 업체 ERP에서 발생한 변경 데이터(CUD)를 중앙 서버와 양방향으로 주고받고, 각 업체는 company_id / server_id 로 구분하여 통합 관리합니다.
업체 ERP A
company_id: COMP-A
server_id: ERP-A01
업체 ERP B
company_id: COMP-B
server_id: ERP-B01

중앙 서버 → 업체 ERP : 정책/동기화/관리 응답
• 등록/수정/삭제(CUD) 발생 시 중앙 API로 변경 내역 전송
• 중앙 서버는 업체별 식별자(company_id / server_id) 기준으로 데이터를 분리 저장 및 통합 관리
