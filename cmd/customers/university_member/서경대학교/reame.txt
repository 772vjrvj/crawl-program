1. 전체 목적

    1-1. 목적

        서경대학교 공개 조직도 페이지를 기준으로
        대학, 대학원, 부속기관, 부설연구소 목록을 수집한다.

        각 조직의 홈페이지 URL에 접속하여 교수 메뉴 URL을 찾고,
        교수명과 이메일을 수집한다.

        일반 구조로 수집하기 어려운 일부 페이지는
        별도 예외 case로 직접 수집한다.

        최종 결과는 원본 엑셀과 정리본 엑셀
        총 2개로 저장한다.


    1-2. 현재 수집 범위

        1) 조직도 목록 수집

        2) 조직별 홈페이지 URL 수집

        3) 교수URL 탐색

        4) 교수URL 안의 교수 이름 / 이메일 수집

        5) 교수URL을 찾지 못한 경우 구분용 빈 row 저장

        6) 교수 목록을 찾지 못한 경우 구분용 빈 row 저장

        7) 교수 수집 후 학부 공지사항에서 행정 이메일 추가 수집

        8) 인성교양대학 / 교양교육연구소 예외 수집

        9) 대학원 조직도 예외 수집

        10) 예술교육원 예외 수집

        11) 원본 엑셀 저장

        12) 이메일 기준 정리본 엑셀 저장


    1-3. 현재 제외 범위

        1) 전화번호는 수집하지 않는다.

        2) 비공개 페이지나 로그인 필요 데이터는 수집하지 않는다.

        3) 고객 납품용 별도 컬럼 재가공은 제외한다.

        4) 이름, 이메일, 직위, 업무 중심으로 저장한다.

        5) 학부 공지사항에서는 @skuniv.ac.kr 이메일만 수집한다.

    4-1. 기준 URL

    4-2. 수집 대상

        TARGET_MIDDLE_NAMES 값에 포함된 중분류만 수집한다.

            대학
            대학원
            부속기관
            부설연구소


    4-3. 조직도 목록 수집 방식

        1) 기준 URL에 접속한다.

        2) class="org-middle-group" 목록을 찾는다.

        3) org-middle-button 텍스트가 수집 대상인지 확인한다.

        4) org-minor-group 안의 org-minor-button 텍스트를 대분류로 저장한다.

        5) org-sub-button의 data-context 값을 읽는다.

        6) data-context 안의 세분류를 중분류 / 소분류로 저장한다.

        7) data-context 안의 홈페이지 값을 URL로 저장한다.


    4-4. 저장 기준

        대분류 = org-minor-button 텍스트
        중분류 = data-context 세분류 또는 버튼 텍스트
        소분류 = data-context 세분류 또는 버튼 텍스트
        URL = data-context 홈페이지


5. 교수URL 찾기

    5-1. 기본 방식

        1) 조직도에서 수집한 홈페이지 URL에 접속한다.

        2) 홈페이지 안의 a[href] 태그를 확인한다.

        3) 교수 메뉴로 판단되는 링크를 찾는다.

        4) 찾은 링크를 교수URL로 저장한다.


    5-2. 교수URL 판단 순서

        1) a 태그 text가 "교수" 이거나 "교수 " 로 시작하는 경우

        2) a 태그 text에 "교수"가 있고 href 또는 class에 professor가 있는 경우

        3) href에 professor가 포함된 경우


6. 일반 교수 정보 수집

    6-1. 수집 영역

        1) 교수URL에 접속한다.

        2) class="panel-wrapper" 목록을 찾는다.

        3) panel-wrapper가 2개 이상이면 두 번째 영역을 우선 사용한다.

        4) 없으면 전체 HTML에서 table row를 찾는다.


    6-2. 교수 row 수집

        1) table tbody tr을 반복한다.

        2) tr 안의 h4 태그에서 이름을 가져온다.

        3) mailto 링크가 있으면 이메일로 저장한다.

        4) mailto가 없으면 row 전체 텍스트에서 이메일 형식을 찾는다.


    6-3. 교수 저장 기준

        직위 = 교수
        업무 = 교육·연구
        이름 = 수집한 교수명
        이메일 = 수집한 이메일
        URL = 조직도 홈페이지 URL
        교수URL = 교수 목록 URL

12. 일반 case 추가 - 학부 공지사항 행정 이메일 수집

    12-1. 추가 목적

        일반 교수 수집이 끝난 뒤
        각 학부 홈페이지 안의 학부 공지사항 메뉴를 확인한다.

        학부 공지사항 게시글 상세 페이지에서
        교수 메일이 아닌 @skuniv.ac.kr 이메일을 찾으면
        행정 이메일로 추가 저장한다.


    12-2. 학부 공지사항 URL 찾기

        1) 각 조직 홈페이지 URL에 다시 접속한다.

        2) a 태그 전체를 확인한다.

        3) a 태그 텍스트에 "학부 공지사항"이 포함되면
           해당 href를 학부 공지사항 URL로 사용한다.

        4) 1차로 찾지 못하면
           텍스트에 "공지사항"이 있고 href에 notice가 포함된 링크를 보조로 사용한다.

        5) 홈페이지에서 찾지 못하면 교수URL 페이지도 보조로 확인한다.

        6) 상대 경로 href는 urljoin으로 절대 URL로 변환한다.


    12-3. 게시판 목록 수집 방식

        1) 학부 공지사항 URL에 접속한다.

        2) table.boardList td.title a[href] 링크를 게시글 상세 링크로 수집한다.

        3) 상세 링크는 document_srl 파라미터가 있거나
           URL 끝이 숫자 형태인 경우를 우선 사용한다.

        4) page 값을 1씩 증가시키면서 다음 페이지를 확인한다.

        5) 신규 게시글 링크가 더 이상 없으면 중지한다.

        6) 무한 반복 방지를 위해 최대 확인 페이지 수는 30페이지로 제한한다.


    12-4. 상세 페이지 이메일 수집 방식

        1) 게시글 상세 URL에 접속한다.

        2) class="viewDocument" 영역을 우선 확인한다.

        3) viewDocument가 없으면 class="boardRead" 영역을 확인한다.

        4) 두 영역이 모두 없으면 상세 페이지 전체에서 확인한다.

        5) 화면 텍스트와 mailto 링크에서 @skuniv.ac.kr 이메일을 찾는다.

        6) 교수 수집 단계에서 이미 수집한 교수 이메일과 같은 이메일은 제외한다.

        7) 교수 메일이 아닌 @skuniv.ac.kr 이메일을 찾는 순간
           해당 학부 공지사항 탐색을 멈추고 다음 학부로 넘어간다.


    12-5. 행정 이메일 저장 기준

        대분류 = 기존 학부 대분류
        중분류 = 기존 학부 중분류
        소분류 = 행정
        직위 = 빈값
        업무 = 행정
        이름 = 빈값
        이메일 = 학부 공지사항 상세 페이지에서 찾은 @skuniv.ac.kr 이메일
        URL = 기존 학부 홈페이지 URL
        교수URL = 이메일을 찾은 공지사항 상세 URL
        상태 = 성공
        메시지 = 학부 공지사항 상세페이지에서 교수 메일이 아닌 @skuniv.ac.kr 이메일을 수집했습니다.


    12-6. 저장하지 않는 경우

        1) 학부 공지사항 URL을 찾지 못한 경우

        2) 게시판 목록에서 상세 링크를 찾지 못한 경우

        3) 상세 페이지들에서 @skuniv.ac.kr 이메일을 찾지 못한 경우

        4) 찾은 @skuniv.ac.kr 이메일이 이미 수집된 교수 이메일과 같은 경우

        위 경우에는 별도 빈 row를 추가하지 않는다.

13. 전체 실행 순서

    13-1. 실행 흐름

        1) Selenium 브라우저 실행

        2) 서경대학교 조직도 접속

        3) 일반 조직 목록 수집

        4) 각 조직 홈페이지 URL 접속

        5) 교수URL 탐색

        6) 교수URL 접속 후 교수 데이터 수집

        7) 교수 수집 후 학부 공지사항 행정 이메일 추가 탐색

        8) 일반 수집 실패 시 구분용 빈 row 저장

        9) 인성교양대학 예외 수집

        10) 대학원 예외 수집

        11) 예술교육원 예외 수집

        12) 상태별 요약 로그 출력

        13) 원본 엑셀 저장

        14) 이메일 빈값 제거 및 이메일 기준 중복 제거

        15) 정리본 엑셀 저장

        16) 브라우저 종료


14. 상태값 기준

    14-1. 성공

        교수명 또는 행정 데이터와 이메일을 정상 수집한 경우이다.


    14-2. 이메일없음

        이름이나 직위 정보는 수집했지만 이메일이 없는 경우이다.


    14-3. 홈페이지없음

        조직도 data-context에 홈페이지 URL이 없는 경우이다.


    14-4. 교수URL없음

        홈페이지에 접속했지만 교수 메뉴 링크를 찾지 못한 경우이다.


    14-5. 교수목록없음

        교수URL은 찾았지만 교수 목록 row를 찾지 못한 경우이다.


    14-6. 페이지실패

        예외 case 페이지나 교수 상세 페이지 접속에 실패한 경우이다.


    14-7. 수집없음

        예외 case 페이지에 접속했지만 수집 가능한 row를 찾지 못한 경우이다.


15. 최종 산출물

    15-1. 코드 파일

    15-2. 설명서 파일

        skuniv_professor_email_notice_final_guide.txt


    15-3. 실행 후 생성되는 엑셀

        1) 서경대학교_교수_URL_이메일_원본.xlsx

        2) 서경대학교_교수_URL_이메일_정리본.xlsx


    15-4. 원본 엑셀 특징

        이메일 없는 행도 남긴다.

        이메일 중복 행도 남긴다.

        자동 수집 실패 후 확인이 필요한 행도 남긴다.


    15-5. 정리본 엑셀 특징

        이메일 빈값 행은 제거한다.

        이메일이 같은 행은 첫 번째 행만 남긴다.

        이메일 비교는 소문자 기준으로 처리한다.

        원본에서 사용할 수 있는 이메일 데이터만 남긴다.
