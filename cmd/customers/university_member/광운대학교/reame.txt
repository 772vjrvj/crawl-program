작성자: ChatGPT
버전: v1.0
대상 사이트: 광운대학교

최종 코드 파일: kw_final_crawler.py


개정 내용:
    1) 광운대학교 lnb-list 메뉴 기준으로 대학 / 대학원 / 특수대학원 목록을 수집한다.

    2) 중분류가 "한눈에 보는 대학"인 row는 제거한다.

    3) 중분류가 "소개"인 경우 최종 중분류 값은 대분류 값으로 대체한다.

    4) 소개 페이지에서는 담당직원 표를 읽어 행정 데이터를 수집한다.

    5) 소개가 아닌 학과 페이지에서는 행정 이메일과 교수진 정보를 수집한다.

    6) 산학협력단 페이지의 popup_write_it 테이블을 별도 case로 수집한다.

    7) 최종 엑셀은 전체 원본 1개, 정리본 1개 총 2개 생성한다.

    8) 정리본은 이메일 빈값 제거 후 이메일 기준으로 중복 제거한다.


1. 전체 목적

    1-1. 목적

        광운대학교 공개 페이지에서 조직 / 학과 / 교수 / 행정 이메일 정보를 수집한다.

        수집한 데이터는 작업자 확인용 전체 원본 엑셀과
        이메일 기준으로 정리된 정리본 엑셀 총 2개로 저장한다.


    1-2. 현재 수집 범위

        1) 광운대학교 lnb-list 메뉴 수집

        2) 분류 / 대분류 / 중분류 / URL 수집

        3) 중분류 "한눈에 보는 대학" 제거

        4) 중분류 "소개"를 대분류명으로 대체

        5) 소개 페이지 담당직원 테이블 수집

        6) 학과 페이지 행정 이메일 수집

        7) 학과 페이지 교수진 URL 수집

        8) 교수진 페이지 교수 이름 / 이메일 수집

        9) 산학협력단 직원 정보 수집

        10) 전체 원본 엑셀 저장

        11) 이메일 빈값 제거 및 이메일 중복 제거 정리본 저장


    1-3. 현재 제외 범위

        1) 전화번호는 저장하지 않는다.

        2) 비공개 페이지나 로그인 필요 데이터는 수집하지 않는다.

        3) 상세 프로필 페이지 내부까지는 들어가지 않는다.

        4) 고객 납품용 별도 컬럼 재가공은 제외한다.


2. 코드 작성 규칙

    2-1. 기본 원칙

        1) 코드는 쉽고 짧고 명확하게 작성한다.

        2) 불필요하게 복잡한 구조는 만들지 않는다.

        3) 타입 체크는 사용하지 않는다.

        4) 예외처리는 최소화한다.

        5) 최신 문법 위주의 어려운 코드는 사용하지 않는다.

        6) 일반 수집과 산학협력단 예외 수집은 함수로 분리한다.


    2-2. 로그 기준

        1) 현재 어느 단계인지 알 수 있게 로그를 출력한다.

        2) 현재 몇 번째 데이터를 처리 중인지 알 수 있게 로그를 출력한다.

        3) 행정 이메일, 교수URL, 교수 수집 건수를 확인할 수 있게 출력한다.

        4) 엑셀 저장 완료 파일명을 출력한다.


    2-3. 속도 기준

        1) Selenium은 headless 모드로 실행한다.

        2) 이미지 로딩은 비활성화한다.

        3) page_load_strategy는 eager로 둔다.

        4) time.sleep은 최소한으로 사용한다.

4. 광운대학교 메뉴 수집

    4-1. 기준 URL

        https://www.kw.ac.kr/ko/univ/glance.jsp


    4-2. 수집 영역

        1) ul class="lnb-list"를 찾는다.

        2) ul.lnb-list 바로 아래 직계 li를 찾는다.

        3) 직계 li 안의 직계 a 텍스트를 분류로 저장한다.

        4) 예시 분류는 대학, 대학원, 특수대학원이다.


    4-3. 대분류 수집

        1) 분류 li 안의 직계 ul을 찾는다.

        2) 해당 ul 안의 직계 li를 반복한다.

        3) 각 li 안의 직계 a 텍스트를 대분류로 저장한다.

        4) 대분류가 "한눈에 보는 대학"이면 저장하지 않는다.


    4-4. 중분류 수집

        1) 대분류 li 안의 직계 ul을 찾는다.

        2) 직계 ul 안의 직계 li를 반복한다.

        3) 각 li 안의 직계 a 텍스트를 중분류로 저장한다.

        4) 각 li 안의 직계 a href를 URL로 저장한다.

        5) 중분류가 "한눈에 보는 대학"이면 저장하지 않는다.

        6) 중분류가 "소개"이면 최종 중분류는 대분류명으로 저장한다.

        7) 소개 여부는 내부 처리용 값으로 따로 보관한다.


5. 소개 페이지 담당직원 수집

    5-1. 판단 기준

        메뉴 수집 당시 원래 중분류가 "소개"였던 row를 소개 페이지로 본다.


    5-2. 수집 영역

        1) section class="h3_contents-block" 목록을 찾는다.

        2) section 안의 h3 텍스트가 "담당직원"인 영역을 찾는다.

        3) 해당 section 안의 table을 수집한다.


    5-3. 테이블 컬럼 기준

        td 1번 = 이름
        td 2번 = 직위
        td 3번 = 업무
        td 6번 = 이메일


    5-4. 저장 기준

        소분류 = 행정
        직위 = 테이블 직위
        업무 = 테이블 담당업무
        이름 = 테이블 성명
        이메일 = 테이블 이메일
        URL = 소개 페이지 URL
        교수URL = 빈값


6. 학과 페이지 행정 이메일 수집

    6-1. 판단 기준

        소개 페이지가 아닌 row는 일반 학과 또는 기관 페이지로 본다.


    6-2. 이메일 수집 기준

        1) span class="ico-circle email" 영역을 먼저 찾는다.

        2) 해당 영역 텍스트에서 이메일 정규식으로 이메일을 추출한다.

        3) 없으면 a href가 mailto로 시작하는 링크를 찾는다.

        4) 찾은 이메일 중 첫 번째 이메일을 저장한다.


    6-3. 저장 기준

        소분류 = 행정
        직위 = 빈값
        업무 = 행정
        이름 = 빈값
        이메일 = 수집한 이메일
        URL = 학과 URL
        교수URL = 빈값


7. 교수진 URL 수집

    7-1. 수집 영역

        1) nav class="tab tab-inc" 안의 a 태그를 찾는다.

        2) a 태그 텍스트에 "교수진"이 포함된 링크를 찾는다.

        3) 찾은 href를 현재 페이지 URL 기준 절대경로로 변환한다.


    7-2. 저장 기준

        교수URL 컬럼에 교수진 URL을 저장한다.


8. 교수진 정보 수집

    8-1. 기준 테이블

        table id="departmentProfessor"를 찾는다.


    8-2. 수집 기준

        1) tbody tr을 반복한다.

        2) td 2번에서 이름을 수집한다.

        3) td 5번에서 이메일을 수집한다.


    8-3. 저장 기준

        소분류 = 중분류
        직위 = 교수
        업무 = 교육·연구
        이름 = 교수 이름
        이메일 = 교수 이메일
        URL = 학과 URL
        교수URL = 교수진 URL


9. 산학협력단 case

    9-1. 기준 URL

        https://iacf.kw.ac.kr/sub01/sub04.php


    9-2. 수집 영역

        1) class="write_it popup_write_it" 영역을 모두 찾는다.

        2) 각 영역 안의 table을 찾는다.

        3) 첫 번째 tr은 헤더이므로 제외한다.

        4) 나머지 tr을 데이터 row로 본다.


    9-3. 테이블 컬럼 기준

        td 1번 = 소속
        td 2번 = 직책(직위)
        td 3번 = 성명
        td 4번 = 담당업무
        td 6번 = EMAIL


    9-4. 저장 기준

        분류 = 산학협력단
        대분류 = 산학협력단
        중분류 = 소속
        소분류 = 행정
        직위 = 직책(직위)
        업무 = 담당업무
        이름 = 성명
        이메일 = EMAIL
        URL = 산학협력단 URL
        교수URL = 빈값


10. 이메일 정리본 생성

    10-1. 정리 순서

        1) 전체 원본 DataFrame을 복사한다.

        2) 이메일 컬럼의 앞뒤 공백을 제거한다.

        3) 이메일이 빈값인 row를 제거한다.

        4) 이메일을 소문자로 변환한 내부 비교 키를 만든다.

        5) 이메일 기준으로 중복 제거한다.

        6) 중복 이메일은 처음 나온 row만 유지한다.

        7) 내부 비교 키는 저장 전에 제거한다.


11. 전체 실행 순서

    11-1. 실행 흐름

        1) Selenium 브라우저 실행

        2) 광운대학교 lnb-list 메뉴 접속

        3) 메뉴 목록 수집

        4) 한눈에 보는 대학 제거

        5) 소개는 대분류명으로 대체

        7) 소개 페이지면 담당직원 수집

        8) 소개가 아니면 행정 이메일 수집

        9) 소개가 아니면 교수진 URL 수집

        10) 교수진 URL이 있으면 교수 정보 수집

        11) 산학협력단 case 별도 수집

        12) 전체 원본 엑셀 저장

        13) 이메일 빈값 제거 및 중복 제거

        14) 정리본 엑셀 저장

        15) 브라우저 종료
