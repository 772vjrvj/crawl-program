# SWU 이메일 수집 최종 설계서

# 0. 기본 기준

## 0-1. 수집 목적
서울여자대학교 공개 페이지에서 대학, 대학원, 연구소, 부속기관, 산학협력단 구성원의 이름과 이메일 정보를 수집한다.

## 0-2. 최종 결과 형태
데이터를 object list 형태로 수집한 뒤 엑셀로 저장한다.

## 0-3. 최종 저장 필드
대분류
중분류
소분류
직위
업무
이름
이메일
URL
홈페이지URL

## 0-4. 제외 필드
전화번호는 수집하지 않는다.

## 0-5. 기본 업무 기준
교수는 업무를 "교육·연구" 로 저장한다.
행정실은 업무를 "행정" 으로 저장한다.
직원, 팀장, 연구원, 조교 등은 페이지에 있는 직위와 업무를 그대로 저장한다.

## 0-6. 코드 작성 규칙
예외처리는 최소화한다.
타입 체크는 하지 않는다.
최신 문법은 사용하지 않는다.
어려운 문법은 사용하지 않는다.
코드는 쉽고 짧고 명확하게 작성한다.

## 0-7. 코드 작성 방향
파이썬으로 작성한다.
Selenium 기준으로 작성한다.
headless 모드로 실행한다.
불필요한 함수 분리는 하지 않는다.
불필요한 검증 코드는 넣지 않는다.
time.sleep은 기본으로 넣지 않는다.
필요한 경우에만 나중에 추가한다.

## 0-8. 엑셀 저장 기준
엑셀 파일명에는 실행 시점의 yyyymmddhhmmss 값을 붙인다.

## 0-9. 홈페이지 접속 기준
학과 홈페이지처럼 외부 도메인으로 이동하는 URL은 접속이 안 될 수 있다.
DNS 오류나 접속 실패가 생길 수 있으므로 이 부분만 필요한 최소 예외처리를 한다.


# 1. 대학

## 1-1. 기준 URL
https://www.swu.ac.kr/www/swuniversity.html

## 1-2. 대학 / 학과 목록 수집

\#main 안의 .section 목록을 반복한다.

각 .section 안에서 .titl1 또는 .titl0 텍스트를 대분류로 사용한다.

경우 1. .titl1 또는 .titl0 안에 a 링크가 바로 있는 경우
해당 a 태그의 text를 대분류, 중분류, 소분류에 모두 넣는다.
해당 a 태그의 href를 URL로 저장한다.

예시
```html
<div class="titl1"><a href="/www/fundaedu_1.html">교양대학</a></div>
```

저장 결과
- 대분류 = 교양대학
- 중분류 = 교양대학
- 소분류 = 교양대학
- URL    = https://www.swu.ac.kr/www/fundaedu_1.html

경우 2. .titl1 또는 .titl0 아래에 ul.col_list0 이 있는 경우
.titl1 또는 .titl0 텍스트를 대분류로 사용한다.
ul.col_list0 li a 텍스트를 중분류와 소분류로 사용한다.
li a의 href를 URL로 저장한다.

예시
- 대분류 = 인문대학
- 중분류 = 국어국문학과
- 소분류 = 국어국문학과
- URL    = https://www.swu.ac.kr/www/liberala_1.html

## 1-3. 전공 탭 처리

학과 페이지에 접속한다.

학과 페이지 안에 class="tabui4" 가 있으면
해당 학과가 여러 전공으로 나뉘어 있는 것으로 본다.

이 경우 tabui4 안의 a 링크를 모두 수집한다.
각 a 링크의 text는 전공명으로 본다.
각 a 링크의 href는 전공 URL로 저장한다.

수집한 전공 URL에 하나씩 접속한다.
각 전공 페이지에서 다시 "교수진" 링크를 찾는다.

전공 탭이 있는 경우 저장 기준
- 대분류 = 기존 대학명
- 중분류 = tabui4 안의 a text
- 소분류 = tabui4 안의 a text
- URL    = 전공 URL

class="tabui4" 가 없으면
현재 학과 페이지에서 바로 "교수진" 링크를 찾는다.

전공 탭이 없는 경우 저장 기준
- 대분류 = 기존 대학명
- 중분류 = 기존 학과명
- 소분류 = 기존 학과명
- URL    = 기존 학과 URL


## 1-4. 홈페이지 URL 수집

학과 또는 전공 페이지 안에서
class="btn btn_xl btn_blue_gray" 인 a 태그를 찾는다.

해당 a 태그의 href를 홈페이지URL로 저장한다.

홈페이지URL이 여러 개 있으면 모두 확인 대상에 포함한다.

## 1-5. 교수진 페이지 찾기

학과 또는 전공 페이지 안에서
class="tabui0" 영역을 찾는다.

tabui0 안의 a 태그 중 text가 "교수진" 인 링크를 찾는다.
찾으면 해당 링크로 이동한다.


## 1-6. 교수 정보 수집

교수진 페이지에서 ul.pro_list를 찾는다.
ul.pro_list 안의 li를 반복한다.

이름은 .CELL.CELL1 안의 .name에서 가져온다.
이메일은 .info 안의 mailto 링크에서 가져온다.

이름에 "교수", "조교수", "부교수", "명예교수", "석좌교수", "특임교수" 같은 문구가 있으면 제거한다.
이름 안의 불필요한 공백도 제거한다.

## 1-7. 교수 데이터 저장 규칙

- 대분류      = 대학명
- 중분류      = 학과명 또는 전공명
- 소분류      = 학과명 또는 전공명
- 직위        = 교수
- 업무        = 교육·연구
- 이름        = 교수 이름
- 이메일      = mailto 이메일
- URL         = 교수진 페이지 URL
- 홈페이지URL = 홈페이지 바로가기 URL

## 1-8. 행정실 이메일 수집

교수 정보 수집이 끝난 뒤 홈페이지URL에 접속한다.

홈페이지 첫 화면의 전체 HTML에서 이메일 형식을 찾는다.
이메일을 찾으면 행정실 데이터로 추가한다.

footer 안에 있을 수도 있고,
다른 위치에 있을 수도 있으므로 페이지 전체에서 찾는다.

## 1-9. 행정실 데이터 저장 규칙

- 대분류      = 대학명
- 중분류      = 학과명 또는 전공명
- 소분류      = 행정실
- 직위        = 빈값
- 업무        = 행정
- 이름        = 빈값
- 이메일      = 홈페이지에서 찾은 이메일

## 2-2. 대학원 교학팀 수집

대학원 교학팀 페이지에서 div.table0.center를 찾는다.
div.table0.center 안의 tbody tr을 반복한다.

각 tr 안에서 td를 순서대로 읽는다.

- 첫 번째 td  = 직위
- 두 번째 td  = 이름
- 세 번째 td  = 전화번호이므로 제외
- 네 번째 td  = 이메일
- 다섯 번째 td = 업무 div가 여러개로 되어 있는데 엔터로 연결

이메일은 네 번째 td 안의 mailto 링크에서 가져온다.

## 2-3. 대학원 교학팀 저장 규칙

- 대분류      = 대학원
- 중분류      = 대학원
- 소분류      = 대학원 교학팀
- 직위        = 첫 번째 td
- 업무        = 다섯 번째 td
- 이름        = 두 번째 td
- 이메일      = 네 번째 td의 mailto 이메일
- URL         = https://www.swu.ac.kr/www/swuprea_58.html
- 홈페이지URL = 빈값

해당 li를 학과 메뉴로 본다.

학과 메뉴 li 안의 바로 아래 ul을 찾는다.
이 ul 안의 li 목록을 기준으로 학과 또는 계열을 판단한다.


## 2-7. 계열이 있는 경우

학과 메뉴 아래 li 안에 다시 ul이 있으면
해당 li는 "계열"로 본다.

구조 예시
ul#gnb
li
- a/span = 학과안내
ul
li
- a/span = 인문계열
ul
- li > a/span = 국어국문학과
- li > a/span = 영어영문학과

처리 방식
1. 학과안내 li 아래의 첫 번째 ul을 찾는다.
2. 그 안의 li를 반복한다.
3. li 안에 하위 ul이 있으면 해당 li의 a/span 텍스트를 계열명으로 본다.
4. 하위 ul 안의 li > a 링크들을 학과 링크로 수집한다.

저장 규칙
- 대분류 = 대학원명 / 계열명
- 중분류 = 학과명
- 소분류 = 학과명
- URL    = 학과 URL

예시
- 대학원명 = 일반대학원
- 계열명   = 인문계열
- 학과명   = 국어국문학과

저장 결과
- 대분류 = 일반대학원 / 인문계열
- 중분류 = 국어국문학과
- 소분류 = 국어국문학과
- URL    = https://www.swu.ac.kr/grd/unlibera_1.html


## 2-8. 계열이 없는 경우

학과 메뉴 아래 li 안에 하위 ul이 없으면
해당 li는 바로 학과 또는 전공으로 본다.

구조 예시
ul#gnb
li
- a/span = 학과소개
ul
- li > a/span = 국어교육전공
- li > a/span = 상담심리전공
- li > a/span = 유아교육전공

처리 방식
1. 학과안내 또는 학과소개 li 아래의 첫 번째 ul을 찾는다.
2. 그 안의 li를 반복한다.
3. li 안에 하위 ul이 없으면 해당 li > a 링크를 학과 링크로 수집한다.
4. li > a 또는 span 텍스트를 학과명 또는 전공명으로 사용한다.

저장 규칙
- 대분류 = 대학원명
- 중분류 = 학과명 또는 전공명
- 소분류 = 학과명 또는 전공명
- URL    = 학과 URL

예시
- 대학원명 = 교육대학원
- 학과명   = 국어교육전공

저장 결과
- 대분류 = 교육대학원
- 중분류 = 국어교육전공
- 소분류 = 국어교육전공
- URL    = https://www.swu.ac.kr/edu/edukor_1.html

그 안의 table.table.table-bordered.table-email을 찾는다.

table 안의 tbody tr을 반복한다.

각 tr 안에서 td를 순서대로 읽는다.

- 첫 번째 td  = 직위
- 두 번째 td  = 이름
- 세 번째 td  = 전화번호이므로 제외
- 네 번째 td  = 이메일
- 다섯 번째 td = 업무

이메일은 네 번째 td 안의 mailto 링크에서 가져온다.

한 td 안에 mailto 링크가 여러 개 있으면
첫 번째 mailto 이메일만 사용한다.

업무는 다섯 번째 td 안의 텍스트를 가져온다.
li가 여러 개 있으면 줄바꿈으로 합쳐서 저장한다.

## 3-13. 산학협력단 저장 규칙

- 대분류      = 산학협력단
- 중분류      = 산학협력단
- 소분류      = table-wrap 안의 title 텍스트
- 직위        = 첫 번째 td
- 업무        = 다섯 번째 td
- 이름        = 두 번째 td
- 이메일      = 네 번째 td의 mailto 이메일

# 4. 예외

## 4-1. 자동 수집 예외

위의 대학, 대학원, 연구소/부속기관 흐름으로 해결되지 않는 데이터는
자동 수집 대상에서 제외한다.

## 4-2. 수작업 처리

자동 수집으로 해결되지 않는 경우
개발자가 해당 홈페이지에 직접 들어가서 수작업으로 확인한다.

## 4-3. 홈페이지 접속 예외

홈페이지 URL이 열리지 않거나 DNS가 잡히지 않는 경우가 있을 수 있다.
이 경우 해당 홈페이지는 수작업 확인 대상으로 남긴다.

## 4-4. 페이지 구조 예외

페이지 구조가 위 기준 태그와 다르면 자동 수집하지 않는다.
개발자가 직접 페이지를 확인하고 필요한 경우 별도 처리한다.


# 5. 총정리

## 5-1. 전체 수집 순서

1. 대학 데이터 수집
2. 대학원 데이터 수집
3. 연구소/부속기관 데이터 수집
4. 산학협력단 데이터 수집
5. 자동 수집이 안 된 예외 데이터는 개발자가 수작업으로 확인
6. 자동 수집 데이터와 수작업 데이터를 합쳐 전체본 엑셀 저장
7. 전체본 엑셀을 작업자가 내부 보관
8. 전체본 엑셀을 기준으로 정리 프로그램을 한 번 더 실행
9. 이메일 빈값과 중복 이메일을 제거한 고객 납품용 최종본 엑셀 생성

## 5-2. 전체본 엑셀

전체본 엑셀은 작업자 보관용 파일이다.

자동 수집 데이터와 개발자가 수작업으로 확인한 데이터를 모두 포함한다.

이메일이 없는 데이터도 남긴다.
이메일이 중복된 데이터도 남긴다.

이 파일은 고객에게 바로 전달하지 않고 작업자가 내부 보관한다.

## 5-3. 고객 납품용 최종본 엑셀

고객에게 전달할 파일은 전체본 엑셀을 기준으로 한 번 더 정리해서 만든다.

정리 프로그램을 다시 실행하여
이메일이 없는 행을 제거한다.

이메일이 동일한 행은
먼저 나온 행만 남기고 뒤에 나온 중복 행은 제거한다.

이렇게 정리된 파일을 고객 납품용 최종본 엑셀로 사용한다.

## 5-4. 중복 제거 기준

이메일 값을 기준으로 중복 여부를 판단한다.

같은 이메일이 여러 번 나오면
먼저 나온 행을 유지한다.

나중에 나온 같은 이메일 행은 고객 납품용 최종본에서 제거한다.

## 5-5. 최종 산출물

전체본 엑셀
작업자 보관용
자동 수집 데이터 포함
수작업 확인 데이터 포함
이메일 빈값 포함
중복 이메일 포함

고객 납품용 최종본 엑셀
고객 전달용
전체본 엑셀을 기준으로 재정리한 파일
이메일 빈값 제거
중복 이메일 제거

## 5-6. 핵심 기준

모든 자동 수집은 공개 페이지 기준으로 진행한다.
전화번호는 수집하지 않는다.
이메일은 mailto 링크 또는 페이지 안의 이메일 형식에서 수집한다.

자동 수집이 어려운 데이터는 개발자가 직접 홈페이지에 들어가서 수작업으로 확인한다.
수작업 확인까지 끝난 전체본 엑셀은 작업자가 보관한다.

고객에게는 전체본을 그대로 전달하지 않는다.
전체본을 기준으로 이메일 빈값과 중복 이메일을 제거한 고객 납품용 최종본을 전달한다.
