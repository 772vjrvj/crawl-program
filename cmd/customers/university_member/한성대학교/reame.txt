1. 전체 목적

    1-1. 목적

        한성대학교 공개 페이지를 기준으로
        대학, 대학원, 산학연구처ㆍ산학협력단의 이름과 이메일 정보를 수집한다.

        대학과 대학원은 학과 / 전공 목록을 수집한 뒤
        각 교수URL에 접속하여 교수 이름과 이메일을 수집한다.

        대학과 대학원은 교수URL 안에서 게시판URL과 공지사항URL도 찾는다.

        교수 목록 수집이 끝난 뒤
        게시판URL 또는 공지사항URL이 있는 학과 / 학부 / 대학원은
        대표로 게시글 이메일 1개를 찾아 행정 데이터로 추가한다.

        산학연구처ㆍ산학협력단은 탭 메뉴의 팀 목록을 수집한 뒤
        각 팀 페이지에서 직원 이름, 직책, 담당업무, 이메일을 수집한다.

        최종 결과는 원본 엑셀과 정리본 엑셀 총 2개로 저장한다.


    1-2. 현재 수집 범위

        1) 대학 학과 / 전공 목록 수집

        2) 대학원 학과 / 전공 목록 수집

        3) 교수URL 수집

        4) 학과 홈페이지 URL 수집

        5) 교수URL 안의 교수 이름 / 이메일 수집

        6) 교수URL 안의 게시판URL 수집

        7) 교수URL 안의 공지사항URL 수집

        8) 게시판 또는 공지사항 상세글에서 행정 이메일 1개 보강 수집

        9) 산학연구처ㆍ산학협력단 팀 목록 수집

        10) 산학연구처ㆍ산학협력단 직원 이름 / 직책 / 담당업무 / 이메일 수집

        11) 원본 엑셀 저장

        12) 이메일 빈값 제거 및 이메일 기준 중복 제거 정리본 저장


    1-3. 현재 제외 범위

        1) 전화번호는 저장하지 않는다.

        2) 로그인 필요한 비공개 데이터는 수집하지 않는다.

        3) 게시판과 공지사항의 모든 이메일을 수집하지 않는다.

        4) 학과 / 학부 / 대학원별 대표 행정 이메일은 최대 1개만 수집한다.


2. 코드 작성 규칙

    2-1. 기본 원칙

        1) 코드는 쉽고 짧고 명확하게 작성한다.

        2) 불필요하게 복잡한 구조는 만들지 않는다.

        3) 불필요한 타입 체크는 넣지 않는다.

        4) 불필요한 예외처리는 최소화한다.

        5) 기존 코드 흐름과 변수명을 최대한 유지한다.

        6) 일반 수집과 예외 수집은 함수로 분리해서 마지막에 합친다.


    2-2. 사용 기술

        Python
        requests
        Selenium
        BeautifulSoup
        pandas
        openpyxl


    2-3. 속도 기준

        3) Selenium은 headless 모드로 실행한다.

        4) Selenium 이미지 로딩은 비활성화한다.

        5) 게시판 페이징처럼 javascript 실행이 필요한 부분만 Selenium을 사용한다.

4. 대학 / 대학원 목록 수집

    4-1. 대학 기준 URL

        https://www.hansung.ac.kr/hansung/6081/subview.do


    4-2. 대학원 기준 URL

        https://www.hansung.ac.kr/hansung/6091/subview.do


    4-3. 수집 영역

        1) class="major-list-title" 목록을 찾는다.

        2) class="major-list-content" 목록을 찾는다.

        3) major-list-title과 major-list-content는 같은 순서로 짝이 맞는 구조이다.


    4-4. 대분류 저장 기준

        major-list-title 안의 h1 텍스트를 대분류로 저장한다.


    4-5. 중분류 저장 기준

        major-list-content 안의 li를 반복한다.

        각 li 안의 class="major-list-content-title" 안에 있는 h1 텍스트를 중분류로 저장한다.


    4-6. URL 저장 기준

        class="link1" 안의 a href는 교수URL로 저장한다.

        class="link2" 안의 a href는 URL로 저장한다.


5. 교수URL 수집

    5-1. 교수 목록 수집 영역

        교수URL에 접속한다.

        class="professor_list" 안의 ul 안의 li 목록을 반복한다.


    5-2. 이름 수집 기준

        li 안의 class="artclInfo" 안에 있는 p 태그에서 이름을 가져온다.

        p 태그 안의 span은 영어 이름이므로 제거한다.


    5-3. 이메일 수집 기준

        li 안의 a[href^="mailto:"] 링크에서 이메일을 가져온다.

        mailto 문자열에서 이메일 형식만 정규식으로 추출한다.


    5-4. 교수 저장 기준

        구분 = 대학 또는 대학원
        대분류 = 목록에서 수집한 대분류
        중분류 = 목록에서 수집한 중분류
        소분류 = 중분류와 동일
        직위 = 교수
        직책 = 빈값
        업무 = 교육·연구
        이름 = 교수명
        이메일 = 교수 이메일
        교수URL = 교수 목록 URL
        URL = 학과 홈페이지 URL


6. 게시판URL / 공지사항URL 수집

    6-1. 수집 시점

        교수URL에 접속했을 때 함께 수집한다.


    6-2. 수집 영역

        class="tab_div div_3" 안의 ul li a 목록을 찾는다.


    6-3. 게시판URL 저장 기준

        a 태그의 텍스트가 "게시판"이면 href를 게시판URL로 저장한다.


    6-4. 공지사항URL 저장 기준

        a 태그의 텍스트가 "공지사항"이면 href를 공지사항URL로 저장한다.


7. 게시판 / 공지사항 행정 이메일 보강

    7-1. 수집 시점

        대학 / 대학원 교수 목록 수집이 모두 끝난 뒤 실행한다.


    7-2. 대상 기준

        게시판URL 또는 공지사항URL이 있는 학과 / 학부 / 대학원을 대상으로 한다.

        같은 학과에 교수 row가 여러 개 있으면 대표 row 1개만 사용한다.


    7-3. 수집 순서

        1) 게시판URL이 있으면 게시판부터 확인한다.

        2) 게시판에서 이메일을 1개 찾으면 즉시 중지한다.

        3) 게시판에서 이메일을 찾지 못하면 공지사항URL을 확인한다.

        4) 공지사항에서 이메일을 1개 찾으면 즉시 중지한다.

        5) 게시판과 공지사항 모두 이메일이 없으면 추가하지 않는다.


    7-4. 페이징 처리

        게시판 또는 공지사항 페이지 안의 class="_paging" 영역을 확인한다.

        class="_totPage" 값이 있으면 전체 페이지 수로 사용한다.

        페이지 이동은 javascript page_link 함수를 Selenium execute_script로 실행한다.


    7-5. 게시글 목록 수집

        table class="board-table" 안의 tbody tr 목록을 반복한다.

        각 tr에서 아래 값을 수집한다.

            td-title 안의 a href = 게시글URL
            td-write 텍스트 = 이름


    7-6. 게시글 상세 수집

        게시글URL에 접속한다.

        class="txt" 영역 안에서 @hansung.kr 이메일 형식만 찾는다.

        class="txt" 영역이 없으면 페이지 전체 텍스트에서 이메일 형식을 찾는다.


    7-7. 제외 기준

        게시글 상세에서 찾은 이메일 중 @hansung.kr 이메일만 추가한다.

        교수 목록에 이미 있는 이메일이면 추가하지 않는다.

        앞에서 이미 행정 이메일로 추가한 이메일이면 추가하지 않는다.


    7-8. 저장 기준

        구분 = 기존 학과의 구분
        대분류 = 기존 학과의 대분류
        중분류 = 기존 학과의 중분류
        소분류 = 행정
        직위 = 빈값
        직책 = 빈값
        업무 = 행정
        이름 = 게시글 작성자
        이메일 = 게시글 상세에서 찾은 이메일
        교수URL = 기존 학과의 교수URL
        게시판URL = 기존 학과의 게시판URL
        공지사항URL = 기존 학과의 공지사항URL
        URL = 게시글URL
        게시글URL = 게시글URL


8. 산학연구처ㆍ산학협력단 수집

    8-1. 기준 URL

        https://www.hansung.ac.kr/rnd/4406/subview.do


    8-2. 기본 분류

        구분 = 산학연구처ㆍ산학협력단
        대분류 = 산학연구처ㆍ산학협력단
        소분류 = 행정


    8-3. 팀 목록 수집

        class="tab_div div_3" 안의 ul li a 목록을 찾는다.

        a 태그 텍스트를 중분류로 저장한다.

        a 태그 href를 URL로 저장한다.


    8-4. 직원 목록 수집

        각 팀 URL에 접속한다.

        class="_wizOdr _prFlList prfl-list" 안의 li 목록을 반복한다.


    8-5. 이름 수집 기준

        class="prfl-name" 안의 strong 텍스트를 이름으로 저장한다.


    8-6. 직책 수집 기준

        직책이 없으면 class="prof-rank" 안의 p 텍스트를 사용한다.


    8-7. 업무 수집 기준

        class="artclInfo" 안에서 dt 텍스트가 "담당업무"인 dl의 dd 값을 업무로 저장한다.


    8-8. 이메일 수집 기준

        class="artclInfo" 안의 mailto 링크에서 이메일을 추출한다.

        mailto가 없으면 EMAIL 항목의 텍스트에서 이메일 형식을 찾는다.
