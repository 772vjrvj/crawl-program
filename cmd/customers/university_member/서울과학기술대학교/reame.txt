4. 대학 / 대학원 조직도 수집

    4-1. 기준 URL

        https://www.seoultech.ac.kr/intro/uvstat/orga


    4-2. 1차 대학 / 대학원 목록 수집

        4-2-1. 기본 수집 영역

            1) 기준 URL에 접속한다.

            2) class가 v4인 li를 찾는다.

            3) class가 v5인 li를 찾는다.

            4) li.v4는 대학 영역이다.

            5) li.v5는 대학원 영역이다.


        4-2-2. 대분류 추출 기준

            1) li.v4 또는 li.v5 안의 ul class="box"를 찾는다.

            2) ul.box 안의 a[href] 태그를 수집한다.

            3) a 태그 안에 br 태그가 있으면 br 앞 텍스트만 사용한다.

            4) a 태그 안에 br 태그가 없으면 a 태그 전체 텍스트를 사용한다.

            5) 추출한 텍스트를 대분류로 사용한다.


        4-2-3. 예시

            공과대학<br>(5개 학과, 2개 학부)

            저장:
                대분류 = 공과대학

            일반대학원

            저장:
                대분류 = 일반대학원


        4-2-4. 1차 저장 기준

            1) 대분류는 추출한 텍스트를 저장한다.

            2) 중분류는 빈값으로 둔다.

            3) 소분류는 빈값으로 둔다.

            4) URL은 a 태그의 href 값을 저장한다.

            5) 상대경로 URL은 기준 도메인을 붙여 절대경로로 변환한다.


5. 2차 학과 / 전공 목록 수집

    5-1. 기본 방식

        1) 1차에서 수집한 URL에 접속한다.

        2) 해당 페이지 안에서 학과 / 전공 목록 영역을 찾는다.

        3) 학과 / 전공 목록은 현재 2가지 구조를 처리한다.


    5-2. 1번 구조

        대상 구조:
            ul class="t15l bg mt10"

        수집 기준:
            1) ul.t15l.bg.mt10 안의 a[href] 태그를 찾는다.

            2) a 태그 텍스트를 중분류로 사용한다.

            3) a 태그 텍스트를 소분류에도 동일하게 사용한다.

            4) 1차에서 수집한 대분류는 그대로 유지한다.

            5) a 태그의 href 값을 URL로 저장한다.


        예시:

            대분류 = 일반대학원
            a 태그 텍스트 = AI공공정책전공

            저장:
                대분류 = 일반대학원
                중분류 = AI공공정책전공
                소분류 = AI공공정책전공


    5-3. 2번 구조

        대상 구조:
            div class="t15l bg mt10"
                div class="oh"
                    a
                    ul
                        li
                            a

        수집 기준:
            1) div.t15l.bg.mt10 안의 div.oh를 찾는다.

            2) div.oh 바로 아래의 a 태그 텍스트를 대분류로 사용한다.

            3) div.oh 안의 ul li a 태그 텍스트를 중분류로 사용한다.

            4) 중분류 텍스트를 소분류에도 동일하게 사용한다.

            5) ul li a 태그의 href 값을 URL로 저장한다.


        예시:

            div.oh 바로 아래 a 텍스트:
                산업공학과

            ul li a 텍스트:
                산업정보시스템전공
                ITM전공

            저장:
                대분류 = 산업공학과
                중분류 = 산업정보시스템전공
                소분류 = 산업정보시스템전공

                대분류 = 산업공학과
                중분류 = ITM전공
                소분류 = ITM전공


    5-4. 2차 목록이 없는 경우

        1) ul.t15l.bg.mt10 구조가 없고

        2) div.t15l.bg.mt10 구조도 없으면

        3) 1차 수집 데이터를 그대로 저장한다.

        4) 이 경우 중분류와 소분류는 빈값으로 유지한다.


6. 교수URL / 교직원URL 수집

    6-1. 기본 방식

        1) 최종 학과 / 전공 URL에 접속한다.

        2) 페이지 안의 a[href] 태그를 찾는다.

        3) a 태그 텍스트를 확인한다.

        4) a 태그 안의 span, strong, b 같은 내부 태그 텍스트도 확인한다.

        5) 지정한 키워드가 포함되어 있으면 해당 href 값을 저장한다.

        6) 상대경로 URL은 현재 페이지 URL 기준으로 절대경로로 변환한다.

        7) javascript:, mailto:, tel: 은 제외한다.


    6-2. 교수URL 탐색 키워드

        교수소개
        교수 소개
        교수진소개
        교수진 소개
        교수진
        전담교수
        참여교수


    6-3. 교직원URL 탐색 키워드

        교직원
        교직원소개
        교직원 소개
        직원소개


    6-4. 저장 기준

        1) 교수 키워드에 맞는 메뉴를 찾으면 교수URL에 저장한다.

        2) 교직원 키워드에 맞는 메뉴를 찾으면 교직원URL에 저장한다.

        3) 찾지 못하면 빈값으로 둔다.


7. 교수 데이터 수집

    7-1. 기본 방식

        1) 각 row의 교수URL을 확인한다.

        2) 교수URL이 있으면 해당 URL에 접속한다.

        3) 교수 목록 구조를 확인한다.

        4) 이름과 이메일을 수집한다.

        5) 모든 교수 row에는 아래 값을 고정으로 넣는다.

            직위 = 교수
            업무 = 교육·연구


    7-2. 교수 데이터 없는 경우 처리

        7-2-1. 교수URL이 없는 경우

            1) 교직원 데이터도 없으면 구분용 빈 row 1개를 추가한다.

            2) 직위, 업무, 이름, 이메일은 빈값으로 저장한다.


        7-2-2. 교수URL은 있지만 교수 목록을 못 찾은 경우

            1) 교직원 데이터도 없으면 구분용 빈 row 1개를 추가한다.

            2) 직위, 업무, 이름, 이메일은 빈값으로 저장한다.

            이름 = 추출한 이름
            이메일 = 추출한 이메일


    9-4. 교직원 3번 케이스

        대상 구조:
            table class="table01"

        조건:
            헤더에 이메일과 담당업무가 포함되어 있어야 한다.

        수집 기준:
            1) 첫 번째 tr에서 th 또는 td를 읽어 헤더로 사용한다.

            2) 헤더 중 이메일 컬럼을 찾는다.

            3) 헤더 중 담당업무 컬럼을 찾는다.

            4) 이름 또는 성명 헤더가 있으면 이름도 저장한다.

            5) 두 번째 tr부터 데이터를 반복 수집한다.

        저장:
            소분류 = 행정
            직위 = 행정
            업무 = 담당업무 컬럼 값 또는 행정
            이름 = 이름 또는 성명 컬럼 값
            이메일 = 이메일 컬럼 값


10. 이름 정리 기준

    10-1. 기본 정리

        이름 텍스트에서 앞뒤 공백을 제거한다.

        여러 개의 공백은 하나로 줄인다.


    10-2. 교수명 제거 대상 단어

        교수
        학과장
        전담
        참여


    10-3. 예시

        김선민 교수
        → 김선민

        김진현 학과장
        → 김진현


11. 이메일 추출 기준

    11-1. 기본 정규식

        [A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}


    11-2. 화면 표시 이메일 우선

        1) box 전체 화면 텍스트를 붙여서 읽는다.

        2) 공백 때문에 분리된 이메일을 보정한다.

        3) 예를 들어 goomj @seoultech.ac.kr 형태는 goomj@seoultech.ac.kr로 보정한다.

        4) E-mail, email, e-mail 같은 라벨은 제거한 뒤 이메일을 찾는다.

        5) null 같은 불필요한 앞 문자열도 제거한 뒤 이메일을 찾는다.

        6) 화면 표시값에서 이메일이 잡히면 그 값을 저장한다.


    11-3. 이메일 앞 불필요 문자열 제거 기준

        1) L02-970-6856E-mailkimjw@seoultech.ac.kr 형태는 kimjw@seoultech.ac.kr만 저장한다.

        2) nullE-mailjykim@kirams.re.kr 형태는 jykim@kirams.re.kr만 저장한다.

        3) E-mailjhhan@seoultech.ac.kr 형태는 jhhan@seoultech.ac.kr만 저장한다.

        4) 정규식 매칭 전에 E-mail 라벨을 공백으로 바꿔 이메일 앞에 붙지 않게 한다.


    11-4. mailto fallback

        1) 화면 표시값에서 이메일을 못 찾으면 a[href^="mailto:"] 태그를 찾는다.

        2) mailto: 문자열을 제거한다.

        3) ?subject= 같은 쿼리스트링이 있으면 제거한다.


12. 중복 처리

    12-1. URL 중복 접속 방지

        같은 교수URL이 여러 row에 반복될 수 있으므로
        professor_cache를 사용해서 한 번 접속한 교수URL은 다시 요청하지 않는다.

        같은 교직원URL이 여러 row에 반복될 수 있으므로
        staff_cache를 사용해서 한 번 접속한 교직원URL은 다시 요청하지 않는다.


    12-2. 수집 중 내부 중복 제거

        한 교수URL 또는 교직원URL 안에서 여러 구조가 동시에 잡히는 경우를 대비해
        직위 + 업무 + 이름 + 이메일 조합으로 중복 제거한다.


    12-3. 최종 정리본 중복 제거

        고객용 정리본에서는 이메일 기준으로 중복 제거한다.

        동일한 이메일이 여러 row에 있으면 처음 나온 row만 남긴다.

        이메일이 빈값인 row는 고객용 정리본에서 제거한다.


13. 최종 저장 기준

    13-1. 원본 저장 기준

        교수 데이터가 있는 경우 교수 1명당 row 1개를 저장한다.

        교직원 데이터가 있는 경우 교직원 1명당 row 1개를 저장한다.

        교수 데이터와 교직원 데이터가 둘 다 없으면 구분용 빈 row 1개를 저장한다.

        이메일 빈값과 중복 이메일은 제거하지 않는다.


    13-2. 고객용 정리본 저장 기준

        원본 데이터에서 이메일이 빈값인 row를 제거한다.

        이메일 기준으로 중복 제거한다.

        같은 이메일이 여러 번 나오면 처음 나온 row만 남긴다.


    13-3. 교수 row 저장 형태

        대분류 = 조직도 기준 대분류
        중분류 = 조직도 기준 중분류
        소분류 = 조직도 기준 소분류
        직위 = 교수
        업무 = 교육·연구
        이름 = 교수명
        이메일 = 교수 이메일
        URL = 학과 / 전공 URL
        교수URL = 교수 목록 URL
        교직원URL = 교직원 목록 URL


    13-4. 교직원 row 저장 형태

        대분류 = 조직도 기준 대분류
        중분류 = 조직도 기준 중분류
        소분류 = 행정
        직위 = 행정
        업무 = 담당업무 또는 행정
        이름 = 교직원명 또는 빈값
        이메일 = 교직원 이메일
        URL = 학과 / 전공 URL
        교수URL = 교수 목록 URL
        교직원URL = 교직원 목록 URL


14. 전체 실행 흐름

    14-1. 전체 수집 흐름

        1) 기준 URL 접속

        2) li.v4, li.v5 수집

        3) 1차 대분류 URL 수집

        4) 각 URL 접속

        5) 학과 / 전공 목록 수집

        6) 각 학과 / 전공 URL에서 교수URL 수집

        7) 각 학과 / 전공 URL에서 교직원URL 수집

        8) 교수URL 접속 후 교수 데이터 수집

        9) 교직원URL 접속 후 교직원 데이터 수집

        10) 수집 데이터가 없으면 빈 row 추가

        11) 원본 엑셀 저장

        12) 이메일 빈값 제거 및 이메일 기준 중복 제거

        13) 고객용 정리본 엑셀 저장
